# Memory Card

一个基于 React 和 Vite 构建的童趣记忆卡牌游戏。当前版本已经从早期的单页配对 Demo，升级为包含启动页、登录/注册页、主页大厅、关卡选择、游戏主界面、背包、设置和结算弹窗的完整多场景前端原型。

## 项目简介

项目以儿童向记忆配对玩法为核心，并围绕“马卡龙色系 + 软萌气泡组件 + 卡通卡面插画”的统一视觉语言进行了全面 UI 改造。

当前版本的主要特点包括：

- 支持多场景界面流转，而不再只是单一游戏页
- 支持多关卡配置，不同关卡拥有不同卡组数量和网格列数
- 卡片背面已由文字卡面升级为 SVG 卡通插画卡面
- 界面整体采用童趣化视觉风格，包含云朵、彩虹、星星和小动物元素
- 支持加载过渡、气泡提示、按钮弹跳反馈等轻量交互动画
- 支持减少动态效果和增强对比度等可访问性设置

## 功能特性

### 游戏玩法

- 基于随机洗牌的记忆配对玩法
- 支持实时统计已完成配对数、星星分数和翻牌次数
- 支持关卡切换后按不同卡组规模重新开局
- 支持配对失败后延时翻回
- 支持配对成功后的匹配态高亮
- 支持通关后的星级结算与再次挑战

### 界面与体验

- 启动页趣味加载动画
- 登录 / 注册页双态切换
- 主页大厅欢迎区、推荐关卡与快捷入口
- 关卡选择页展示难度、奖励和目标步数
- 游戏战斗页包含顶部统计区、左侧提示区和中央卡片棋盘
- 背包面板与设置面板均以气泡弹窗形式展示
- 通关后展示软萌结算弹窗，可选择重玩或返回大厅

### 视觉系统

- 明亮柔和的马卡龙配色
- 大圆角、软阴影、气泡按钮和玩具感面板
- 统一的卡通 SVG 牌面资源
- 响应式布局，兼顾桌面端与移动端

## 核心模块说明

### `src/App.jsx`

应用根组件，负责组织整个多场景体验流程，包括：

- 启动页到登录页的初始流转
- 登录 / 注册状态切换
- 大厅、关卡页与游戏页切换
- 背包与设置面板开关
- 加载层、底部气泡提示与结算弹窗控制
- 当前关卡、玩家信息和设置项状态管理

### `src/hooks/useGameLogic.js`

这是项目的核心游戏逻辑模块，负责统一管理对局状态和配对规则，包括：

- 根据关卡配置生成卡组
- 使用 Fisher-Yates 算法随机打乱卡片顺序
- 记录当前轮次中第一张已翻开的卡片
- 判断两张卡片是否属于同一配对
- 累计分数、翻牌次数与完成进度
- 配对失败时通过定时器延迟翻回
- 在重置或组件卸载时清理定时器

### `src/components/Card.jsx`

负责渲染单张卡片，并根据状态切换卡面表现：

- 正面显示统一的“翻翻看”提示
- 背面显示对应的 SVG 卡通插画
- 已匹配卡片展示匹配态高亮
- 支持等待回翻阶段的禁用态

### `src/components/GameHeader.jsx`

负责展示游戏主界面顶部信息，包括：

- 当前关卡名称和难度
- 已完成配对数、星星分数与翻牌次数
- 进度条
- 返回大厅、打开背包、打开设置与重新整理卡片等操作入口

### `src/components/WinMessage.jsx`

负责在通关后展示结算信息，包括：

- 当前关卡名称
- 星级评价
- 本局配对成绩
- 奖励提示
- 再玩一次和返回大厅按钮

### `src/index.css`

负责定义整套童趣化样式系统，包括：

- 全局配色变量和字体
- 气泡按钮、卡片和面板的统一视觉规则
- 卡片翻面与轻量动效
- 加载动画与提示弹层样式
- 桌面端与移动端响应式布局

## 游戏规则

1. 进入大厅后选择喜欢的关卡。
2. 开始挑战后点击任意卡片将其翻开。
3. 再点击第二张卡片进行配对。
4. 如果两张卡片属于同一主题，则保留翻开状态并记为一次成功配对。
5. 如果两张卡片不匹配，则会在短暂展示后自动翻回。
6. 当所有卡片都完成配对时，进入通关结算。

## 当前界面表现

- 整体采用浅粉到浅蓝的柔和背景渐变
- 卡片背面为卡通插画卡面，不再使用单调文本主视觉
- 所有主要操作按钮均带有轻微弹跳反馈
- 加载层使用“小动物跳动 + 进度条”的趣味表现
- 提示与设置面板采用软萌气泡模态风格
- 小屏设备下会自动压缩布局、按钮宽度和卡面尺寸
